Sohu — The Tech You Must Speak To Fluently
Core thesis: By burning the transformer architecture directly into silicon, Sohu reaches ~90%+ FLOP utilization on transformer inference vs. ~30% for an H100. The bet is that transformers are dominant enough (and stable enough) that giving up generality is worth a 10–20× perf/$ uplift. One 8×Sohu server = 160 H100s on Llama-70B inference, per Etched's claim.

Process

4 nm

TSMC 4nm, same node as Nvidia B200. Reticle-limit die — yield is a major manufacturing concern.

Memory

144 GB

HBM3E (8 stacks, Rambus PHY). Holds 70B-param model in FP8 with KV-cache headroom.

HBM BW

~4.8 TB/s

Enough to keep decode phase off the memory wall — beats H100's 3.35 TB/s.

Throughput

500K tok/s

Marketing claim on 8×Sohu server, Llama-70B, FP8 (2048 in / 128 out). No third-party benchmarks yet.

⚙️

What's "Hardcoded"

QKV projection — dedicated MAC arrays per head
Multi-head attention + softmax — fixed pipeline, no kernel scheduler
FFN with GELU / SiLU activations — baked-in nonlinearity blocks
Layer normalization — hardwired
FP8 native compute — no FP16/BF16 fallback (huge silicon savings)
No general-purpose compute units. Data flows through a fixed pipeline — almost no control plane on die.
⚠️

Why It's Risky (and Where MFG Lives)

Architecture lock-in: if Mamba / SSMs / diffusion supplant transformers in inference, Sohu is dead silicon. Mitigated by GPT-5 / Claude / Gemini still being transformer-based.
Reticle-limit die on 4nm: yield is the single biggest cost lever. Wafer cost ~$20K → defect density and bin-out strategy directly determine $/chip.
HBM3E supply: SK Hynix / Micron-constrained — same scarcity Nvidia fights. Etched must lock allocation early.
CoWoS packaging: TSMC advanced packaging is capacity-constrained. Sohu competes with Nvidia/AMD for slots.
System assembly: 8×Sohu server box (board, cooling, power, networking) — this is where your Apple/Square CM mgmt experience plugs in directly.
